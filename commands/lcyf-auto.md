# /lcyf-auto

## 元数据
- **使用 Agents:** 所有Agents（智能选择）
- **遵循 Rules:** 00-总则

## 命令说明
智能分析用户意图，自动选择并执行最合适的命令

## 使用方式

```
/lcyf-auto <任务描述>
```

## 工作原理

### 1. 意图识别

根据关键词和上下文分析用户意图：

```javascript
意图分类表
=========
新功能开发 → /lcyf-new-feature
  关键词: "添加"、"新增"、"实现"、"开发"

Bug修复 → /lcyf-build-fix
  关键词: "修复"、"fix"、"bug"、"问题"

规划设计 → /lcyf-new-feature (规划阶段)
  关键词: "规划"、"设计"、"方案"、"架构"

代码质量 → /lcyf-code-review
  关键词: "审查"、"检查"、"优化"、"质量"

安全检查 → /lcyf-security-scan
  关键词: "安全"、"漏洞"、"风险"、"加密"

测试相关 → /lcyf-tdd
  关键词: "测试"、"单元测试"、"覆盖率"

构建问题 → /lcyf-build-fix
  关键词: "构建失败"、"编译错误"、"打包失败"

模块问题 → /lcyf-module-check
  关键词: "模块"、"依赖"、"循环依赖"

数据库相关 → /lcyf-db-review
  关键词: "数据库"、"SQL"、"表设计"、"索引"

API相关 → /lcyf-api-review
  关键词: "API"、"接口"、"RESTful"

重构清理 → /lcyf-refactor
  关键词: "重构"、"清理"、"优化代码"

验证检查 → /lcyf-verify
  关键词: "验证"、"检验"、"确认"、"ready"
```

### 2. 上下文分析

结合当前项目状态：

```javascript
上下文因素
=========
1. Git状态
   - 有未提交更改 → 考虑代码审查/验证
   - 分支名包含"feature" → 新功能开发
   - 分支名包含"fix" → Bug修复

2. 最近错误
   - 构建失败 → 构建修复
   - 测试失败 → TDD/测试覆盖

3. 文件类型
   - Controller文件 → API审查
   - Mapper.xml文件 → 数据库审查
   - pom.xml修改 → 模块检查
```

### 3. 智能决策

```
决策优先级
=========
1. 安全 > 功能 > 性能 > 文档
2. 修复 > 新增
3. 测试 > 实现
```

## 决策流程图

```
用户输入
  ↓
关键词提取
  ↓
意图匹配 ─────→ 多个匹配？ ─→ 显示选项让用户确认
  ↓              ↓
单一匹配        用户选择
  ↓              ↓
上下文验证 ←────┘
  ↓
显示选择的命令
  ↓
用户确认
  ↓
执行命令
```

## 决策规则库

### 规则1: 安全优先
```
IF 关键词包含 ["密码", "加密", "token", "安全"]
THEN 推荐 /lcyf-security-scan
```

### 规则2: 构建失败立即修复
```
IF 上次构建失败
AND 用户提到 "编译" 或 "构建"
THEN 强制推荐 /lcyf-build-fix
```

### 规则3: 新功能需要规划
```
IF 任务描述 > 50字
AND 关键词包含 "新功能"
THEN 推荐先执行规划阶段
```

### 规则4: 提交前必须验证
```
IF Git有未提交更改
AND 用户提到 "提交" 或 "commit"
THEN 推荐先执行 /lcyf-verify pre-commit
```

### 规则5: 模块变更需要检查
```
IF 修改了 pom.xml
OR 修改了模块间接口
THEN 推荐执行 /lcyf-module-check
```

## 交互模式

### 自动模式（默认）
显示推荐命令，等待用户确认

### 静默模式
```
/lcyf-auto --silent <任务描述>
```
不需要确认，直接执行最匹配的命令

### 解释模式
```
/lcyf-auto --explain <任务描述>
```
只显示决策过程，不执行命令

### 批量模式
```
/lcyf-auto --batch <任务1> <任务2> <任务3>
```
一次处理多个任务

## 注意事项

- **首次使用需要学习**: 前几次可能需要手动确认
- **复杂任务建议分解**: 非常复杂的任务拆分为多步
- **危险操作需要确认**: 删除、重构等操作会强制确认
- **保持描述清晰**: 越清晰的描述，识别越准确
