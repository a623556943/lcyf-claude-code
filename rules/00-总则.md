# 00-总则

## 概述

本规则集为 lcyf-claude-code 的核心规范，所有开发活动必须遵循。

## 基本原则

### 1. 质量优先

- 代码质量高于开发速度
- 所有代码必须通过审查

### 2. 安全第一

- 禁止硬编码敏感信息
- 所有输入必须验证
- 遵循最小权限原则

### 3. 可维护性

- 代码应该易于理解
- 避免过度工程化
- 保持代码简洁

### 4. 团队协作

- 遵循统一的编码规范
- 使用有意义的提交信息
- 及时同步团队知识

## 强制要求

### 必须遵守

| 要求 | 说明 |
|------|------|
| 代码审查 | 所有代码变更必须经过审查 |
| 构建通过 | 提交前必须确保构建成功 |
| 安全检查 | 敏感代码必须进行安全检查 |

### 禁止事项

| 禁止 | 原因 |
|------|------|
| 硬编码密钥 | 安全风险 |
| System.out.println | 使用日志框架 |
| 循环依赖 | 架构问题 |

## 工作流程

```
需求分析 → 设计评审 → 开发
```

## 规则优先级

当规则冲突时，按以下优先级执行：

1. 编码风格（02-编码风格.md）
2. Java 编码规范（06-Java编码规范.md）
3. 其他规范

## 豁免申请

特殊情况需要豁免规则时：

1. 在代码中添加注释说明原因
2. 获得团队负责人批准
3. 记录到项目文档
