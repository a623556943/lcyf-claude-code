# 快速开始

## 5分钟上手

### 1. 安装

```bash
# 全局安装
npm install -g lcyf-claude-code

# 验证安装
lcyf --version
# 输出: lcyf-claude-code v2.0.0
```

### 2. 初始化项目

```bash
cd /path/to/your/java/project
lcyf init
```

按提示填写项目信息：
- 项目名称
- 基础包名（如 com.lcyf.cloud）
- 作者信息

### 3. 开始使用

在 Claude Code 中输入命令：

```
/lcyf-新功能 添加用户导出功能
```

## 常用命令

### 功能开发

| 命令 | 说明 |
|------|------|
| `/lcyf-新功能 <描述>` | 完整的新功能开发流程 |
| `/lcyf-tdd <描述>` | 测试驱动开发 |
| `/lcyf-构建修复` | 修复编译错误 |

### 代码审查

| 命令 | 说明 |
|------|------|
| `/lcyf-代码审查` | 全面代码审查 |

| `/lcyf-API审查` | API设计审查 |
| `/lcyf-数据库审查` | 数据库设计审查 |

### 模块管理

| 命令 | 说明 |

| 命令 | 说明 |
|------|------|

## 典型工作流程

### 开发新功能

```
1. /lcyf-新功能 添加用户批量导入功能

   自动执行:
   ├── 需求分析 (规划专家)
   ├── 架构设计 (架构专家)
   ├── TDD开发 (测试专家 + Java开发专家)
   ├── 代码审查 (代码审查专家)
   ├── 安全扫描 (安全审查专家)
   └── 知识沉淀 (学习代理)
```

### 修复Bug

```
1. /lcyf-tdd 修复用户列表分页错误

   自动执行:
   ├── 编写失败测试
   ├── 修复代码
   ├── 确保测试通过
   └── 验证覆盖率
```

### 提交前检查

```
1. /lcyf-验证

   自动执行:
   ├── 构建检查
   ├── 测试检查
   ├── 代码规范检查
   └── 安全扫描
```

## 项目结构

初始化后的项目结构：

```
your-project/
├── .lcyf/                    # lcyf配置和知识库
│   ├── config.json          # 项目配置
│   ├── learned-patterns/    # 学习到的模式
│   ├── team-conventions/    # 团队约定
│   └── instincts/           # 本能规则
├── src/
│   └── main/java/
│       └── com/example/
│           ├── adapter/     # 适配层
│           ├── biz/         # 业务层
│           └── api/         # API层
└── pom.xml
```
