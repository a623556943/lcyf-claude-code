# 安装指南

## 系统要求

| 组件 | 说明 |
|------|------|
| Claude Code CLI | AI 编码助手 |
| Git | 版本控制工具 |

---

## 安装步骤

### 1. 进入项目目录

```bash
cd /path/to/your/java/project
```

### 2. 克隆配置

```bash
git clone https://github.com/a623556943/lcyf-claude-code.git .claude
```

### 3. 验证安装

检查目录结构：

```bash
ls .claude/
# 应该看到: agents/ commands/ rules/ skills/ templates/
```

---

## 目录结构

安装后：

```
your-project/
├── .claude/                    # lcyf-claude-code 配置
│   ├── agents/
│   ├── commands/
│   ├── rules/
│   ├── skills/
│   ├── templates/
│   └── CLAUDE.md
├── build-parent/doc/knowledge/ # 知识库（自动创建）
├── src/
├── pom.xml
└── .gitignore
```

## 更新配置

```bash
cd .claude
git pull
```

---

## Git 配置

### 不提交 .claude 到仓库

```bash
echo ".claude/" >> .gitignore
```

### 提交 .claude 给团队共享

```bash
git add .claude/
git commit -m "feat: 添加 lcyf-claude-code 配置"
git push
```

**团队成员初始化**：

```bash
git clone <your-repo>
cd <your-project>
# .claude 目录已包含，直接使用
```

## 卸载

```bash
rm -rf .claude/
```

## 常见问题

### Q: 命令不生效？

检查 `.claude/` 目录是否存在且包含完整文件：

```bash
ls .claude/
# 应该有: agents/ commands/ rules/ skills/ templates/
```

### Q: 如何更新到最新版本？

```bash
cd .claude
git pull
```

### Q: 可以在多个项目中使用吗？

可以。在每个项目目录下执行安装步骤即可。

### Q: 知识库存储在哪里？

`build-parent/doc/knowledge/` 目录，执行命令时自动创建。
