# 项目级配置说明

## 什么是项目级配置？

将 lcyf-claude-code 配置放在项目的 `.claude/` 目录下，而不是全局 `~/.claude/` 目录。

## 优势

| 特性 | 项目级配置 | 全局配置 |
|------|-----------|---------|
| 团队共享 | ✅ | ❌ |
| 版本控制 | ✅ | ❌ |
| 独立环境 | ✅ | ❌ |
| CI/CD 集成 | ✅ | ❌ |

---

## 安装

```bash
cd your-project
git clone https://github.com/a623556943/lcyf-claude-code.git .claude
```

## 目录结构

```
your-project/
├── .claude/                    # lcyf-claude-code 配置
│   ├── agents/
│   ├── commands/
│   ├── rules/
│   ├── skills/
│   └── templates/
├── build-parent/doc/knowledge/ # 知识库（自动创建）
├── src/
└── pom.xml
```

---

## Git 配置

### 不提交到仓库

```bash
echo ".claude/" >> .gitignore
```

### 提交给团队共享（推荐）

```bash
git add .claude/
git commit -m "feat: 添加 lcyf-claude-code 配置"
git push
```

团队成员克隆项目后自动包含配置。

---

## 更新配置

```bash
cd .claude
git pull
```

---

## 自定义规范

在 `.claude/rules/` 下添加或修改规则文件：

```bash
# 编辑 Java 规范
vim .claude/rules/06-Java编码规范.md
```

项目级规范优先级 > 全局规范。

---

## CI/CD 集成

`.claude/` 目录包含在项目中，CI 环境自动可用。

**GitHub Actions 示例**：

```yaml
- name: Checkout
  uses: actions/checkout@v3

- name: Build
  run: mvn clean install
  # .claude/ 配置已包含
```

---

## 常见问题

### Q: 如何在多个项目中使用？

在每个项目目录下执行安装命令即可。

### Q: 如何自定义模板？

编辑 `.claude/templates/` 下的模板文件。

### Q: 知识库存储在哪？

`build-parent/doc/knowledge/` 目录，自动创建。
